---

## Название (пример)

**Автоматизация: чат-бот на n8n + память (Qdrant) + подключение LLM (OpenAI‑совместимый API)**

---

## Кратко для клиента

Соберу для вас **сценарий в n8n**: вход по вебхуку или из мессенджера (по договорённости), ответ **языковой моделью**, опционально **семантическая «память»** в Qdrant (чтобы бот помнил контекст пользователя). Развёртывание в **Docker** — отдельный этап или включено в пакет, обсуждаем.

---

## Что вы получите на выходе

1. **Готовый workflow (JSON)** для импорта в n8n или настройки на вашем сервере.
2. **Инструкцию по запуску**: переменные окружения, проверка здоровья, тестовый запрос.
3. **Демо-поток**: тестовый вебхук + пример тела запроса.
4. При необходимости — **доработка** под Telegram / другой канал (отдельным этапом).

---

## Технологии

- **n8n** — оркестрация, вебхуки, HTTP-запросы к API.
- **Qdrant** — векторная БД для поиска и хранения фрагментов «памяти» (в расширенной конфигурации).
- **LLM** через OpenAI‑совместимый API (OpenAI, совместимые прокси, локальные шлюзы — по вашему ключу и политике).

---

## Ограничения и честность (важно для Kwork)

- Ключи API и квоты — **на стороне заказчика**; без ключа рабочий ответ модели недоступен.
- В витрине **SonBot** вебхук ассистента отдаёт **синхронный JSON** (ответ модели в том же запросе). На нестандартных установках n8n при редких сбоях `Respond to Webhook` отладка всё равно доступна в **Executions**.
- Персональные данные: при работе с реальными пользователями нужна **политика хранения** и согласие; я помогаю с технической схемой, не с юридическим консалтингом.

## Пакеты (пример ценности — цены свои)

| Пакет | Состав |
|--------|--------|
| **Старт** | Один workflow: вебхук → нормализация входа → вызов LLM; инструкция + тест. |
| **Память** | + Qdrant: запись/чтение контекста, коллекция, пример апсерта и поиска. |
| **Канал** | + Telegram (или другой) бот, связка с тем же сценарием. |

---

## Что нужно от заказчика

- Доступ к n8n (или VPS/Docker), либо согласие на передачу JSON для импорта.
- `OPENAI_API_KEY`, при необходимости `OPENAI_BASE_URL` и имя модели.
- Требования к тону ответов и ограничения (что бот **не** должен делать).
